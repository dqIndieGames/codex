### 分支与风险

- 默认关闭路径：用户不改配置时，analytics、feedback、log_db、rollout batch flush、app-server 通知合并都不应启用。
- 显式开启路径：用户写入 `config.toml` 后，各能力独立恢复。
- rollout 风险：批量 flush 可能降低极端崩溃时最后少量历史落盘概率，因此默认关闭，开启后保留批次末尾和 shutdown 等强制 flush 点。
- app-server 通知合并风险：输出刷新会从逐条变成小段，不能延迟错误、完成、输入等关键反馈。
- schema 风险：用户禁止本地 `just write-config-schema`，因此 schema 只能人工同步并用静态检查兜底。
- release 风险：本地不编译，所有编译问题只能通过 GitHub Actions 暴露并迭代修复。

### 用户影响

- 默认用户不会看到新按钮或弹窗。
- 默认运行更轻、更安静：少统计、少反馈日志、少 sqlite 日志入库。
- 开启通知合并后，客户端输出刷新可能变成小批量更新。
- 开启 rollout 批量 flush 后，磁盘 I/O 降低，但极端崩溃时最后少量记录风险提高。
