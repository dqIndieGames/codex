## 冻结决策

### 1. Scope

- in-scope 只覆盖 `Responses` 主链：
  - `/responses` HTTP request retry 链
  - `Responses` websocket connect / reconnect 链
  - `Responses` websocket request send 链
  - `Responses` websocket read/event failure 链
  - sampling stream disconnect / idle timeout / wrapped websocket error 导致的 reconnect warn
- out-of-scope：
  - `/models`、compact、memories、其他非 `Responses` 端点
  - payload 级 debug trace
  - 普通终态错误日志
  - rollout / history / prompt / replay / message history 协议
  - `codex-rs/otel/src/events/shared.rs` 的全局宏语义

### 2. suppress 真值

- request：
  - `attempt == 0` 且不会继续 retry：允许发 request OTEL log/trace
  - `attempt == 0` 且会继续 retry：禁止发 request OTEL log/trace
  - `attempt > 0`：无论最终成功还是失败，都禁止发 request OTEL log/trace
- websocket connect：
  - 已处于 `retry_chain_active`：禁止发 connect OTEL log/trace
  - 首轮 connect 失败后仍会继续 `Responses` 主链 retry/reconnect：禁止发 connect OTEL log/trace
  - 首轮成功或首轮非 retryable terminal failure：保留
- websocket request：
  - 已处于 `retry_chain_active`：禁止发 websocket request OTEL log/trace
  - 首轮 send failure 后仍会继续 `Responses` 主链 retry/reconnect：禁止发 websocket request OTEL log/trace
  - 首轮成功或首轮非 retryable terminal failure：保留
- websocket event：
  - 只 suppress 会驱动同一条 `Responses` 主链继续 retry/reconnect 的失败型 event
  - 至少覆盖 close、timeout、read error、`response.failed`、parse error、`response.completed` 前流关闭
  - 正常成功 event、普通 timing event、不会继续 retry 的终态 event 不顺手静音
- reconnect warn：
  - 只 suppress `stream disconnected - retrying sampling request ...`
  - 其他非 retry warn 与终态错误日志不动

### 3. 保留边界

- metrics 不 suppress。
- `request_retry_notifier` 只负责 UI retry 状态链，不新增日志职责。
- `will_retry = true`、`Reconnecting... N`、`additional_details`、历史区不落脏记录这些现有行为全部保留。
- `401` / auth recovery 若继续同一主链，则该次 request / websocket 中间态 OTEL log/trace 按中间噪音 suppress；auth recovery 最终失败仍沿现有终态错误路径处理。
- 不允许通过改坏 `codex-rs/otel/src/events/shared.rs` 来全局一刀切静音所有 OTEL 事件。

### 1. 请求层的 `401/auth recovery` suppress 边界没有真正闭环
- 严重度：高
